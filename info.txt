wsgi, asgi - Файлы, обеспечивающие корректное подключение к серверу.
(asgi - более новый стандарт, wsgi - более старый)

settings - в данном файле описываются все глобальные настройки данного проекта

Переменные, внутри него:

* BASE_DIR - отвечает за полный путь к данному проекту
* SECRET_KEY - необходимо изменить при выгрузке сайта в сеть (или же реализовать его в .env файле)
* DEBUG - отвечает за показ ошибок на самом сайте (при загрузке на сервер изменяем её на false)
* ALLOWED_HOSTS - можем организовать внутри данного списка домены, на которых можно размещать данный вебсайт
* INSTALLED_APPS - список, который хранит все те приложения, которые установлены в данном проекте
    1. admin - обеспечение панели администратора
    2. sessions - работа с сессиями
    3. staticfiles - работа со статическими файлами
    4. messages - работа с сообщениями
* MIDDLEWARE - список, содержащий различное промежуточное ПО, плагины/классы/библиотеки,
    обеспечивающие безопасность, работу с сессиями, поддержку с CSRF токенами, авторизацию и т.д.
* ROOT_URLCONF - в нём объявлен файл основной urls, с которым мы будем работать
* TEMPLATES - шаблоны, которые можно использовать внутри данного проекта
* WSGI_APPLICATION - файл wsgi, который используется
* DATABASES - с какой БД работает данный проект (sqllite,postgreSQL,mySQL and etc...)
* STATIC_URL - директория файла со статикой

urls - внутри данного файла можем отслеживать различные url адреса данного проекта

python manage.py startapp "name" - создание нового приложения внутри данного проекта
    + admin.py - конкретно какие отношения (таблицы) должны выводиться в панеле администратора
    + apps.py - можно описать какие - либо настройки для данного приложения - приложения name
    + mosels.py - в нём реализуются классы, на основе которых будут созданы таблицы в БД
    + tests.py - можно реализовать тесты для данного приложения (UNIT тесты и т.д...)
    + views.py - отвечает за те методы, которые будут вызваны при переходе пользователя на определенную страницу

создание доп. Директории в папке templates необходимо для того, чтобы при объединении всех приложений не произошла замена одних файлов на других

если стили автоматически не применяются, необходимо на странице сайта сделать следующее:
Просмотреть код -> Зажать кнопку "Перезагрузить страницу" -> "Очистка кэша и жесткая перезагрузка" -> Готово, теперь стили должны примениться.
